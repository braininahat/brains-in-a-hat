# Design Context: Grant UI

## Target Audience
OSS developers using brains-in-a-hat. Needs to be self-explanatory and polished for strangers.

## Use Cases
- Ask about repo architecture, get an interactive visualization
- Compare approaches visually
- Understand what changed in a codebase via timelines/graphs
- Any question where "seeing" beats "reading"

## Brand Personality
**Academic warmth** — like a well-designed textbook or lecture hall. Warm, inviting, intellectually rigorous. The feeling of a great professor who makes complex things obvious.

## Overdrive Direction
**Control room** — streaming feels alive. See Grant thinking in real-time: progress stages (reading repo, analyzing, generating viz), the feeling of watching a mind work. Not cold/NASA — warm and intellectually engaging.

## Key Principle
The interface is for OSS developers who stumble on the project. First impression matters. The "wow" should be: "I can see it thinking, and then the answer appears as a visualization."
